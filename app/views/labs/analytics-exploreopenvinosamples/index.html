Introduction

The OpenVINO™ toolkit is a comprehensive toolkit for quickly developing applications and solutions that emulate human vision. Based on Convolutional Neural Networks (CNNs), the toolkit extends computer vision workloads across Intel hardware, maximizing performance.

The OpenVINO™ toolkit:

  • Enables the CNN-based deep learning inference on the edge.
  • Supports heterogeneous execution across Intel's computer vision accelerators, using a common API for the CPU, Intel® Integrated Graphics, Intel® Movidius™ Neural Compute Stick, and FPGA.However, we are not covering Intel® Movidius™ and FPGA in this lab.
  • Speeds time-to-market through an easy-to-use library of computer vision functions and pre-optimized kernels.
  • Includes optimized calls for Computer Vision standards, including OpenCV, OpenCL, and OpenVX.

Take a look at the OpenVINO™ Homepage.

Pre-requisites
  • System Requirements
    • Only the CPU, Intel® Integrated Graphics, and Intel® Movidius™ Neural Compute Stick options are supported for the Windows installation.
    • Linux is required for using the FPGA or Intel® Movidius™, Myriad™2 VPU options
  • Processors
    • 6th - 8th Generation Intel® Core™
    • Intel® Xeon™ v5 family,Intel® Xeon™ v5 family
  • Operating System
    • Microsoft Windows* 10 64-bit

Learning Outcome

By the end of this module, the participant is expected to understand the Intel® OpenVINO™ toolkit, installation folder structure and performance difference between CPU and GPU

  • OpenVINO™ Toolkit installer

    OpenVINO™ toolkit by default installs at C:\Intel\computer_vision_sdk_2018.1.265\

  • OpenVINO™ Toolkit Samples applications

    OpenVINO™ samples is made available in C:\Intel\computer_vision_sdk_2018.1.265 \deployment_tools\inferenceengine\samples\build\samples.sln

  • OpenVINO™ Toolkit Documentation directory

    C:\Intel\computer_vision_sdk_2018.1.265\deployment_tools\documentation

  • OpenVINO™ Toolkit pre-trained models directory

    C:\Intel\computer_vision_sdk_2018.1.265\deployment_tools\intel_models

Go to C:\Intel\computer_vision_sdk_2018.1.265 \deployment_tools\inferenceengine and understand the Package Contents

  • bin folder has 64-bit runtime libraries for OpenVINO™ samples

    E.g., cLDNN64.dll , cLDNNPlugin.dll etc

  • doc folder has documentation for OpenVINO™ samples like classification, object detection, interactive_face_detection etc.

  • include folder has several header files required for developing application using OpenVINO™ toolkit

  • lib folder has 64-bit plugin library like inference_engine and libiomp5md, are useful for video applications

    • inferenceengine.dll  is the software library used for loading inference engines plugins for CPU,GPU,etc.

    • libiomp5md.dll  is Intel® OMP runtime library used for developing application using OpenMP.

Security Barrier Camera Sample

  • Description

    Show Vehicle Detection followed by Vehicle Attributes and License Plate Recognition applied on top of Vehicle Detection. The Vehicle Attributes execution barrier reports the general vehicle attributes, like the vehicle type and color, where type is something like car, van or bus. The application reads command line parameters and loads the speciic models. The Vehicle/License-Plate Detection model is required, and others are optional.

  • Running the Application

    • Go to C:\Intel > computer_vision_sdk_2018.1.265 >deployment_tools > inferenceengine > samples>build > security_barrier_camera_sample folder

    • Open samples.sln file and build security_barrier_camera_sample.cpp sample. after successful build security_barrier_camera_sample.exe file will be available inside bin\intel64\Debug folder

    • Run the application with -h to see the various options available for running the application.

    • Here we are not using all parameters, participants are expected to explore multiple options.

  • To run the sample application with CPU, execute the below command.

    C:\Intel\computer_vision_sdk_2018.1.265\deployment_tools\inference_engine\bin\intel64\Debug\security_barrier_camera_sample.exe -i C:\Intel\computer_vision_sdk_2018.1.265\deployment_tools\demo\car_1.bmp -m C:\Intel\computer_vision_sdk_2018.1.265\deployment_tools\intel_models\vehicle-license-plate-detection-barrier-0007\FP32\vehicle-license-plate-detection-barrier-0007.xml -m_va C:\Intel\computer_vision_sdk_2018.1.265\deployment_tools\intel_models\vehicle-attributes-recognition-barrier-0010\FP32\vehicle-attributes-recognition-barrier-0010.xml -d CPU

    Output & Performance:

    • The output uses OpenCV to display the resulting frame with detections rendered as bounding boxes and text with vehicle attributes, license plate, detection time and fps. The inference has been done using pre-trained model on CPU
    • Next we will repeat the exercise with GPU and observe the performance.
    • Press ESC button to terminate


  • Case 2: To run the sample application on GPU. Execute the below command

    C:\Intel\computer_vision_sdk_2018.1.265\deployment_tools\inference_engine\bin\intel64\Debug\security_barrier_camera_sample.exe -i C:\Intel\computer_vision_sdk_2018.1.265\deployment_tools\demo\car_1.bmp -m C:\Intel\computer_vision_sdk_2018.1.265\deployment_tools\intel_models\vehicle-license-plate-detection-barrier-0007\FP32\vehicle-license-plate-detection-barrier-0007.xml -m_va C:\Intel\computer_vision_sdk_2018.1.265\deployment_tools\intel_models\vehicle-attributes-recognition-barrier-0010\FP32\vehicle-attributes-recognition-barrier-0010.xml -d GPU

    Output & Performance:

    • The output uses OpenCV to display the resulting frame with detections rendered as bounding boxes and text with vehicle attributes, license plate, detection time and fps. Compared to CPU acceleration, GPU acceleration is better in terms of detection time and fps.
    • Press ESC button to terminate


    • Intel® OpenVINO toolkit libraries, header and sample code files.
