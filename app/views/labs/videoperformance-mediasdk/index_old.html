  • Open the sample_decode Solution in MS Visual Studio 2015,

    build the solution successfully without any errors

  • If there are errors, rectify them. Known rectification methods involve:
    • Right clicking the solution in Visual studio solution explorer and click retarget solution
    • If warning is treated as error, right click both projects one by one, right click on each project, select properties, click C/C++.

      Set "Treat Warnings As Errors" to No(/WX). Then click Apply and OK.

    • Microsoft left a few things out of their MSVT package. If there is a fatal error cannot run 'rc.exe', 'cl.exe', 'ml.exe'

      Then copy rc.exe, rcdll.dll, cl.exe, ml.exe from

      C:\Program Files (x86)\Windows Kits\8.1\bin\x86 to

      C:\Program Files (x86)\Microsoft Visual Studio 14.0\VC\bin

  • Goto _build > x64 > Debug folder and run as follows:
  • Open the _build > x64 > Debug path in command prompt and test Software decode-
    • sample_decode.exe h264 -i C:\input.h264 –r -sw
  • Open the _Build > x64 path in command prompt and test Hardware decode-
    • sample_decode.exe h264 -i C:\input.h264 –r -hw
  • Open the task manager and compare CPU performance in each of the above cases.
  • Just type sample_decode.exe and explore other command line switches:
    -o : output to file in RAW formats
    -r: Present
    -f : change rendering frame rate
    -w and –h : changing the resolution
    etc
